# UpDown Backend

A Python backend template built with **FastAPI**, **InfluxDB Core 3**, and a
placeholder **Machine-Learning** module вЂ” ready to serve a **Vite** frontend.

---

## Tech stack

| Layer | Technology |
|-------|-----------|
| Web framework | [FastAPI](https://fastapi.tiangolo.com/) |
| ASGI server | [Uvicorn](https://www.uvicorn.org/) |
| Time-series DB | [InfluxDB Core 3](https://docs.influxdata.com/influxdb3/core/) |
| ML (placeholder) | [scikit-learn](https://scikit-learn.org/) / NumPy |
| Config management | [pydantic-settings](https://docs.pydantic.dev/latest/concepts/pydantic_settings/) |
| Tests | pytest + httpx |

## Quick start

### 1 вЂ” Clone & configure

```bash
cp .env.example .env
# Edit .env and fill in INFLUXDB_TOKEN etc.
```

### 2 вЂ” Install dependencies

```bash
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements-dev.txt
```

### 3 вЂ” Run the dev server

```bash
uvicorn app.main:app --reload
```

The API is now available at <http://localhost:8000>.  
Interactive docs: <http://localhost:8000/docs>

### 4 вЂ” Docker Compose (full stack)

```bash
docker compose up --build
```

This starts both the FastAPI service and an InfluxDB Core 3 instance.

## API overview

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/v1/health` | Liveness probe |
| `POST` | `/api/v1/data/write` | Write a data point to InfluxDB |
| `POST` | `/api/v1/data/query` | Execute a SQL query against InfluxDB |
| `POST` | `/api/v1/predict` | Run the ML model on a feature vector |

Full interactive documentation is auto-generated by FastAPI at `/docs`.

## CORS & Vite frontend

By default the backend allows requests from `http://localhost:5173` (Vite's
default dev-server port).  To add more origins, edit `CORS_ORIGINS` in your
`.env` file:

```dotenv
CORS_ORIGINS=["http://localhost:5173","https://my-production-domain.com"]
```

## Adding a real ML model

1. Train or load your model in `app/ml/model.py`.
2. Replace `DummyModel` with a class that exposes a `predict(features)` method.
3. Update `PredictionRequest` / `PredictionResponse` in `app/models/schemas.py`
   if you need richer input/output types.
